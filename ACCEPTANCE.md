# Criterios de aceptacion

## Negocio y contenido

- [ ] La accion principal es visible en todas las paginas relevantes.
- [ ] El contenido coincide con `CONTENT.md`.
- [ ] No hay textos ficticios ni datos comerciales inventados.

## Diseno y experiencia

- [ ] La implementacion respeta `DESIGN.md`.
- [ ] Se revisaron desktop, tablet y mobile.
- [ ] En mobile, los margenes y espaciados verticales son compactos y no hay grandes bloques vacios.
- [ ] No hay etiquetas decorativas por encima de titulares, excepto las aprobadas expresamente en contenido o diseno.
- [ ] Menu, formularios, enlaces y CTAs funcionan.
- [ ] Hay estados de carga, exito y error cuando corresponden.
- [ ] El boton flotante de WhatsApp funciona, es accesible y no bloquea contenido ni otros CTAs.

## Calidad y accesibilidad

- [ ] Los controles se pueden usar con teclado.
- [ ] El foco es visible y los formularios tienen labels.
- [ ] Imagenes con significado tienen texto alternativo.
- [ ] No hay enlaces rotos ni errores visibles.

## Publicacion

- [ ] Cada pagina tiene title de pestana y meta descripcion unicos; favicon, Open Graph, sitemap y robots revisados.
- [ ] El footer incluye informacion comercial aprobada, contacto, navegacion interna util y enlaces legales cuando aplican.
- [ ] El preview de Vercel fue aprobado.
- [ ] Variables de entorno revisadas para Preview y Production.
- [ ] Dominio, formularios y analitica probados antes de comunicar el lanzamiento.
